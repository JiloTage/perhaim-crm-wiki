タスクリスト画面仕様

最終更新: 2026-03-19
Per Heim CRM v3.2 タスクリスト画面（/crm/tasks）のワイヤーフレームと詳細仕様。ヘッダーフィルタ、タスク行コンポーネント、タスク作成モーダル、ソート・フィルタ仕様、インタラクションフローを図解・HTMLモックアップ・Mermaidダイアグラムで視覚的に定義する。

本記事はタスクリスト画面のワイヤーフレームと視覚仕様を中心に記述する。 フロントエンド全体のアーキテクチャはCRMフロントエンド設計を、 バックエンドAPIの詳細はCRM API設計を、 ダッシュボードのタスクウィジェットはダッシュボード画面仕様を参照のこと。

フィルタ・ソート仕様

タスクリスト上部に配置されるフィルタバーにより、表示対象を絞り込む。デフォルトソートは優先度降順→期限昇順→作成日時降順の3段階。

ソート仕様

デフォルトのソート順序（複合ソート）:

  1. 優先度（降順）: urgent → high → normal → low
  2. 期限（昇順）: 近い期限が上位
  3. 作成日時（降順）: 新しいタスクが上位
GET /v1/tenant/crm/tasks
  ?status=pending,in_progress
  &assignee_id=user_001
  &customer_id=cust_123
  &priority=urgent,high
  &task_type=follow_up
  &due_date=2026-03-19
  &sort=priority_desc,due_at_asc,created_at_desc
  &limit=50
  &offset=0

インタラクション仕様

タスク完了処理はチェックボックスクリックから始まる楽観的UI更新パターンを採用し、API通信の成否に応じてUIを確定またはロールバックする。

タスク完了フロー

sequenceDiagram
    participant User as 営業担当
    participant UI as TaskListView
    participant API as CRM API
    participant DB as Firestore

    User->>UI: チェックボックスをクリック
    UI->>UI: 楽観的UI更新
（打ち消し線 + 薄グレー表示）
    UI->>API: POST /v1/tenant/crm/tasks/{id}/complete
    API->>DB: status = "completed"
completed_at = now()

    alt 成功
        DB-->>API: 200 OK
        API-->>UI: { status: "completed" }
        UI->>UI: 完了セクションに移動
（スライドアニメーション）
    else 失敗
        DB-->>API: 500 Error
        API-->>UI: Error Response
        UI->>UI: ロールバック
（元のスタイルに戻す）
        UI->>User: エラートースト表示
「タスクの完了に失敗しました」
    end

操作一覧

操作トリガー挙動
タスク完了 チェックボックスクリック 楽観的UI更新 → POST /tasks/{id}/complete → 成功: 完了セクションに移動 / 失敗: ロールバック
顧客カードへ遷移 顧客名リンククリック /crm/customers/:customerId へ画面遷移
タスク作成 「＋ タスク作成」ボタン 作成モーダル表示 → フォーム入力 → POST /tasks → リストに追加
フィルタ変更 フィルタドロップダウン変更 クエリパラメータ更新 → GET /tasks → リスト再描画

ドラッグ&ドロップ操作フロー

カードのドラッグ&ドロップによるフェーズ遷移は、楽観的更新パターンを採用する。UIを即座に更新し、バックグラウンドでAPIを呼び出す。失敗時はロールバックする。詳細な実装はカンバン実装詳細を参照。

sequenceDiagram participant U as ユーザー participant UI as カンバンUI participant S as 状態管理
(Pinia Store) participant M as 遷移理由モーダル participant API as CRM API U->>UI: カードをドラッグ開始 UI->>UI: カード半透明+回転(3deg) UI->>UI: ドロップ先カラムをハイライト U->>UI: カードをドロップ UI->>S: 楽観的更新(moveCard) S->>UI: UI即時反映(カード移動) alt 遷移理由が必要なフェーズ変更 UI->>M: 遷移理由モーダル表示 M->>U: 理由入力を要求 U->>M: 理由を入力して確定 M->>S: 遷移理由をセット end S->>API: PATCH /customers/{id}/phase Note over API: {phase, reason, assigneeId} alt API成功(200) API-->>S: 更新後データ返却 S->>UI: 確定表示(アニメーション) else API失敗(4xx/5xx) API-->>S: エラー返却 S->>S: ロールバック(元のカラムに戻す) S->>UI: カード元位置に復帰 UI->>U: エラートースト表示 end

遷移理由モーダル仕様

以下のフェーズ遷移では理由入力が必須となる。

遷移パターン理由入力備考
前方遷移（フェーズアップ）任意スキップ可能
後方遷移（フェーズダウン）必須差し戻し理由の記録
2段階以上のスキップ必須スキップ理由の記録
「失注」への遷移必須失注理由の分析用データ

操作状態遷移

カンバンボードの操作状態は以下のステートマシンで管理される。@dnd-kitの各イベントと連動して状態が遷移する。

stateDiagram-v2 [*] --> idle: 初期状態 idle --> dragging: onDragStart
カードをつかむ dragging --> idle: onDragCancel
ESCキー / 範囲外ドロップ dragging --> dropped: onDragEnd
カラムにドロップ dropped --> confirming: 遷移理由が必要 dropped --> saving: 遷移理由不要 confirming --> saving: 理由入力して確定 confirming --> idle: モーダルキャンセル
→ ロールバック saving --> success: API 200 OK saving --> error: API 4xx/5xx success --> idle: 完了アニメーション後 error --> idle: ロールバック完了
→ エラートースト表示 state idle { [*] --> waiting waiting: カード操作可能 } state dragging { [*] --> holding holding: カード半透明+回転 holding: ドロップ先ハイライト } state saving { [*] --> api_call api_call: PATCH /customers/{id}/phase api_call: 楽観的UI更新済み }

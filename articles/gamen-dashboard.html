ダッシュボード画面仕様

最終更新: 2026-03-19
Per Heim CRM v3.2 ダッシュボード画面（/crm）のワイヤーフレームと詳細仕様。KPIカード、デュアルファネル、本日のタスク、直近の反響の4セクション構成を、図解・HTMLモックアップ・Mermaidダイアグラムで視覚的に定義する。

本記事はダッシュボード画面のワイヤーフレームと視覚仕様を中心に記述する。 フロントエンド全体のアーキテクチャはCRMフロントエンド設計を、 バックエンドAPIの詳細はCRM API設計を、 分析機能の全体像はCRM分析・レポート設計を参照のこと。

KPIカード仕様

カード名値算出ロジックトレンドカラー
新規反響 12件 直近7日間の新規反響数 ↑ +3 #1976d2

完了率 68% hearing_done / (hearing_active + hearing_pending + hearing_done) ↑ +5% #00796b

データソース

KPIデータは以下のAPIエンドポイントから取得する:

GET /v1/tenant/crm/analytics/dashboard

Response:
{
  "new_inquiries":     { "value": 12, "trend": +3,  "trend_pct": 33.3  },
  "hearing_active":    { "value": 5,  "trend": -1,  "trend_pct": -16.7 },
  "visit_reserved":    { "value": 3,  "trend": +2,  "trend_pct": 200.0 },
  "contracted":        { "value": 1,  "trend": 0,   "trend_pct": 0.0   },
  "completion_rate":   { "value": 68, "trend": +5,  "trend_pct": 7.9   }
}

トレンド表示ルール:

  • trend > 0 → ↑ 緑（ポジティブ指標の場合）
  • trend < 0 → ↓ 赤（ネガティブ変動の場合）
  • trend === 0 → → グレー

インタラクション仕様

sequenceDiagram
    participant User as 営業担当
    participant UI as Dashboard UI
    participant API as CRM API

    User->>UI: チェックボックスをクリック
    UI->>UI: 楽観的UI更新（打ち消し線表示）
    UI->>API: PATCH /v1/tenant/crm/tasks/:id {status: "done"}
    API-->>UI: 200 OK
    UI->>UI: 完了アニメーション

    User->>UI: タスク行をクリック
    UI->>UI: /crm/customers/:customerId へ遷移
操作挙動
チェックボックスクリックタスク完了API呼び出し。楽観的UI更新で即座に打ち消し線+薄表示。失敗時はロールバック
タスク行クリック紐づく顧客カード画面（/crm/customers/:customerId）へ遷移
「もっと見る」クリックタスク一覧画面（/crm/tasks）へ遷移

レスポンシブ対応

ダッシュボード画面は3つのブレークポイントで段階的にレイアウトを変化させる。

ブレークポイント定義

デバイス幅KPIカード中段レイアウト反響リスト
デスクトップ ≥ 1024px 5列横並び 2カラム（ファネル + タスク） フル表示
タブレット 768 - 1023px 3列 + 2列の2行 1カラム縦積み ソース名省略
モバイル < 768px 横スクロール 1カラム縦積み コンパクト表示
